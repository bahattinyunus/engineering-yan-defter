# Contributing Guidelines

We welcome contributions! Please follow these steps to get started:

1. **Fork the repository** and clone it to your local machine.
2. **Create a new branch** for your feature or bug fix:
   ```bash
   git checkout -b my-feature
   ```
3. **Make your changes**. Ensure code is formatted and follows the style of existing files.
4. **Add tests** if applicable and run existing tests to ensure nothing breaks.
5. **Commit your changes** with a clear commit message:
   ```bash
   git commit -m "Add XYZ feature"
   ```
6. **Push the branch** to your fork and open a Pull Request against the `main` branch.
7. **Fill out the PR template** and describe the changes clearly.

### Code of Conduct
Please also review the `CODE_OF_CONDUCT.md` file for community expectations.

Thank you for helping improve this repository!
